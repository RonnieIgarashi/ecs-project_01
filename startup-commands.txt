== ECS Project 01 — 起動手順 ==

[前提]
- terraform コマンド: WSL で実行
- docker / aws コマンド: PowerShell で実行
- 作業ディレクトリ(WSL): ~/programming/ECS/ecs-project_01/terraform
- ECR URL: 335142514101.dkr.ecr.ap-northeast-1.amazonaws.com/ecs-project-01

---

## 1. ECR だけ先に作る（WSL）
terraform apply -target=aws_ecr_repository.app

## 2. ECR ログイン（PowerShell）
$password = aws ecr get-login-password --region ap-northeast-1
docker login --username AWS --password $password 335142514101.dkr.ecr.ap-northeast-1.amazonaws.com

## 3. Build & Push（PowerShell）
docker build --platform linux/amd64 -t ecs-project-01 C:\Users\ronni\programming\ECS\ecs-project_01
docker tag ecs-project-01:latest 335142514101.dkr.ecr.ap-northeast-1.amazonaws.com/ecs-project-01:latest
docker push 335142514101.dkr.ecr.ap-northeast-1.amazonaws.com/ecs-project-01:latest

## 4. フル apply（WSL）
terraform apply

## 5. 動作確認後 destroy（WSL）
terraform destroy
※ ECR はイメージが残るためエラーになるが無視してOK（次回再利用可能）

---

## コード修正後の再デプロイ（インフラ起動中）

# Build & Push（PowerShell） → 手順3と同じ
# 強制再デプロイ（PowerShell）
aws ecs update-service --cluster ecs-project-01-cluster --service ecs-project-01-service --force-new-deployment --region ap-northeast-1
